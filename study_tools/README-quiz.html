e-dumper Quiz

A standalone, offline-capable quiz tool for practising multiple-choice questions. Runs entirely in the browser — no server, no account, no installation required. Supports single-answer and multi-select questions with optional per-question explanations.

Quick Start

  1. Open e-dumper-quiz.html in any modern web browser
  2. Click the ☰ toggle to open the sidebar
  3. Select a Mode — Study for immediate feedback or Exam for a timed, no-feedback test
  4. In Study mode, also choose an Answers style (Highlight or Show Answer)
  5. Click 📂 Import Questions and select a .txt quiz file
  6. Questions load automatically — the sidebar collapses and studying begins

The sidebar is collapsed by default. Click ☰ to open it and ☰ again to close it. On mobile it is pinned to the top of the screen.

Stats bar

The stats bar above the question panel shows live counts of ✓ Correct, ✗ Wrong, and ⚑ Flagged questions, updated after every submission.

Study Modes

Select the mode from the Mode dropdown in the sidebar. Both sub-modes display a ✓/✗ feedback panel below the question after answering.

Study — Highlight (default)

The correct option is highlighted in green before you answer — useful to see the right answer while studying. For single-answer questions, clicking any option auto-submits immediately. For multi-select questions a Submit Answer button appears instead.

Study — Show Answer

Options are plain until you interact. Click an option to select it, then click Show Answer to reveal the correct answer without being graded. If you want to be graded on a multi-select question, select all options you think are correct and click Submit Answer.

Tip: Multi-select questions (those with more than one correct answer) are automatically detected. The Submit Answer button appears for these regardless of Answers mode — select all correct options before submitting.

Exam Mode & Review

Switch to Exam in the Mode dropdown to simulate a real test. The Answers sub-mode dropdown is hidden and no feedback is given during the exam.

During the exam

  • A Submit Answer button is always shown (for both single-answer and multi-select questions).
  • After submitting, options are disabled. Your selected answer is shown highlighted but the correct answer is not revealed — no ✓/✗ feedback panel appears.
  • You can navigate freely with ← Previous and Next →, and flag questions with ⚑ Flag or M.
  • Click the red End Exam button in the navigation bar at any time to go to the review screen.

Exam Review — list view

After clicking End Exam, a summary line and a compact scrollable list appear. Each row shows the question number, a text preview, and a colour-coded status badge:

BadgeBorder colourMeaning
✓ CorrectGreenYou selected the right answer(s).
✗ IncorrectRedYou selected a wrong answer.
— SkippedGreyYou navigated away without submitting an answer.

Click Back to Exam to return to the question navigator.

Exam Review — detail view

Click any row to open the full detail view for that question. It shows:

  • The status badge and full question text
  • All options with correct answers highlighted green and your wrong pick highlighted red
  • The explanation inline (if the question has one) — no button needed in review

Use ← Previous and Next → to step through questions in order without returning to the list, or click ← Back to List to return to the summary.

Note: Wrong answers from an Exam session can be exported via 📥 Export Wrong Answers in the sidebar. The exported .txt file uses the same format as the original quiz and can be re-imported directly for a focused retry.

Flagged Questions

Flag any question using the ⚑ Flag for Review button or the M key. Flagged questions display an amber FLAGGED badge next to the question text.

  • The navigation bar shows the live count: ⚑ Review Flagged (N). The button is disabled when no questions are flagged.
  • Clicking Review Flagged opens a modal listing all flagged questions with a short preview. Click any item to jump directly to that question and close the modal.
  • Flags survive shuffling — the flag travels with the question object regardless of deck order.
  • Flags are saved to localStorage and restored when you reopen the app.
  • Use 📥 Export Flagged in the sidebar to save flagged questions as a .txt file for a focused follow-up session.

Explanation

Individual questions can carry an optional explanation that is shown after answering.

During study / exam

When a question has an explanation and has been answered, a 💡 Explanation button appears in the action bar. Clicking it opens a popup panel containing the explanation text. The popup closes automatically after 20 seconds or can be dismissed early with the × button. Navigating to another question also closes it.

In Exam Review

When viewing a question's detail in the Exam Review, the explanation (if present) is always shown inline — no button needed.

Adding explanations to your file

Add an EXPLANATION: line immediately after the Answer: line for any question. It is optional — questions without it are unaffected.

Answer: B
EXPLANATION: Paris has been the capital of France since the 10th century.

File Format

Quiz files must be plain text (.txt) with one question block per section. Blank lines between elements are optional but recommended for readability.

Question 1. What is the capital of France?

A) London
B) Paris
C) Berlin
D) Madrid

Answer: B
EXPLANATION: Paris has been the capital of France since the 10th century.

Question 2. Which of the following are prime numbers?

A) 2
B) 4
C) 5
D) 6

Answer: A,C

Element reference

ElementSupported formatsRequired
Question headerQuestion N., Question N:, Question:Yes
OptionsA), A., or A prefix — letters A through FYes
Single answerAnswer: BYes
Multi-select answerAnswer: A,C — comma-separated, no spaces requiredYes (for multi-select)
ExplanationEXPLANATION: your text here — must follow the Answer lineNo

Rules

  • Questions with more than one correct answer in Answer: are automatically treated as multi-select.
  • Blank lines are ignored by the parser — they are purely cosmetic.
  • The EXPLANATION: keyword is case-insensitive (Explanation: also works).
  • The last question in the file is included only if it has at least one option and an Answer: line.
  • Lines that do not match any recognised pattern are silently skipped.

Data Persistence

Progress is automatically saved to localStorage after every action and restored the next time you open the file in the same browser.

What is saved

  • The full question list — text, options, correct answers, and explanations
  • Each question's answered state, selected answers, and flag
  • Current question position
  • Correct count, wrong-answer list, and total answered
  • Study / Exam mode and Answers mode selection

What is not saved

  • Shuffle order — questions reload in the saved order; re-shuffle manually if desired
  • The Exam review screen state — revisit it by clicking End Exam again
  • The explanation popup state — popup is always closed on load

Click 🗑️ Clear Progress in the sidebar to erase all saved data and start fresh.

Note: localStorage is scoped to the page origin (protocol + hostname + port). If you open the file from a different folder path or a different local server port, previously saved state will not be available.

Running the App

Double-click e-dumper-quiz.html to open it directly in a browser — no server needed. To serve it locally:

# Python
python -m http.server 8000 -d e-dumper/

# Node / npx
npx serve e-dumper/

Then open http://localhost:8000/e-dumper-quiz.html in your browser.

Browser & Device Support

Works in all modern browsers on desktop and mobile:

  • Chrome / Edge 90+
  • Firefox 90+
  • Safari 14+ (iOS and macOS)

The app is fully responsive. On mobile the sidebar becomes a collapsible panel pinned to the top of the screen. All touch targets meet the 44 px minimum for comfortable tapping, the 300 ms tap delay is removed, and iOS Safari auto-zoom on inputs is suppressed.
